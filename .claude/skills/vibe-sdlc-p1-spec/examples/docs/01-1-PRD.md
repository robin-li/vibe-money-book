# 01-1 產品需求文件 (PRD)

> **專案名稱**：Easy Dine — 小型餐廳點餐系統
> **版本**：v0.2
> **最後更新**：2026-03-12

---

## 1. 產品概述

Easy Dine 是一套適用於小型餐廳的數位點餐系統。顧客透過掃描餐桌上的 QR Code 進入點餐頁面，瀏覽分類菜單、將餐點加入購物車並提交訂單。餐廳後台可管理菜單、查看與處理訂單。

### 1.1 目標使用者

| 角色 | 說明 |
|------|------|
| 顧客 (Customer) | 餐廳內用餐的客人，透過手機掃碼點餐 |
| 店員 (Staff) | 餐廳工作人員，負責查看訂單、管理出餐 |
| 管理員 (Admin) | 餐廳管理者，負責菜單管理、系統設定 |

### 1.2 核心目標

- 顧客可自助完成點餐流程，無需等待服務員
- 餐廳可即時接收訂單，減少溝通錯誤
- 簡化菜單維護流程

## 2. 功能需求

### 2.1 顧客端（Mobile Web）

#### PRD-F-001：QR Code 掃碼入口

- 每張餐桌貼有專屬 QR Code，編碼內容為點餐頁面 URL（含桌號參數）
- 顧客用手機掃碼後，直接在瀏覽器中開啟點餐頁面，無需下載 App
- 頁面頂部顯示當前桌號，讓顧客確認自己的座位

#### PRD-F-002：分類菜單瀏覽

- 菜單依分類呈現（如：主食、小菜、飲料、甜點等）
- 每個菜品顯示：名稱、圖片、價格、簡短描述
- 支援分類標籤切換，方便快速瀏覽
- 已售完的菜品標示「售完」且不可加入購物車

#### PRD-F-003：購物車功能

- 顧客可將菜品加入購物車，並指定數量
- 購物車內可調整數量或移除品項
- 每個品項可加入備註（如「不要香菜」、「少辣」）
- 顯示各品項小計與訂單總金額
- 購物車資料在頁面關閉前暫存於瀏覽器端

#### PRD-F-004：訂單提交

- 顧客在購物車頁面確認後，點擊「送出訂單」
- 提交後顯示訂單確認頁，包含訂單編號、品項明細、總金額
- 訂單提交後不可由顧客自行修改（需請店員協助）

#### PRD-F-005：加點功能

- 同一桌號在用餐期間可多次掃碼提交新訂單（加點）
- 每次加點視為獨立訂單，但後台可依桌號彙整

#### PRD-F-006：顧客端列印結帳單

- 訂單確認頁提供「列印結帳單」按鈕
- 結帳單內容：桌號、所有訂單品項彙總、各品項數量與小計、總金額
- 同一桌位的多筆訂單（含加點）彙整為一張結帳單
- 顧客持結帳單至櫃檯結帳付款
- 透過瀏覽器列印功能實現，須提供列印友好的排版樣式

#### PRD-F-007：推薦菜品展示

- 點餐頁面頂部設置「推薦」專區，展示管理員設定的推薦菜品
- 推薦菜品以醒目樣式呈現（如標籤、橫幅），吸引顧客注意
- 點擊推薦菜品可直接加入購物車

### 2.2 後台管理端（Web）

#### PRD-F-010：管理員登入

- 後台需帳號密碼登入
- 初期僅需單一管理員角色（不區分 Staff / Admin 權限）

#### PRD-F-011：菜單管理

- 新增菜品：名稱、分類、價格、描述、圖片
- 編輯菜品資訊
- 刪除菜品（軟刪除，已有訂單引用的菜品不可物理刪除）
- 設定菜品狀態：上架 / 下架 / 售完
- 設定推薦菜品：可勾選菜品為「推薦」，推薦菜品將顯示於顧客端推薦專區
- 管理菜單分類（新增、編輯、排序、刪除）

#### PRD-F-012：訂單查看

- 依時間倒序顯示所有訂單
- 可依桌號、訂單狀態篩選
- 訂單詳情：桌號、品項明細、數量、金額、下單時間、訂單狀態

#### PRD-F-013：訂單狀態管理

- 訂單狀態流轉：`待確認` → `準備中` → `已出餐` → `已完成`
- 店員可手動更新訂單狀態
- 支援取消訂單（狀態變為 `已取消`）

#### PRD-F-015：後台列印出餐單

- 訂單詳情頁提供「列印出餐單」按鈕
- 出餐單內容：訂單編號、桌號、各品項名稱、數量、備註
- 出餐單不顯示金額（僅供廚房出餐使用）
- 透過瀏覽器列印功能實現，提供列印友好排版

#### PRD-F-014：桌位管理

- 設定餐廳桌位數量與桌號
- 可產生各桌位對應的 QR Code（供列印張貼）
- 桌位狀態：空桌 / 用餐中 / 待清理
- 可手動將桌位重置為「空桌」（代表該桌結束用餐）
- 記錄桌位入座時間（首次掃碼或手動設定），顯示用餐時長
- 桌位列表顯示各桌目前用餐時長，方便商家追蹤

## 4. 資料模型概要

### 4.1 菜單分類 (Category)

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | Long | 主鍵 |
| name | String | 分類名稱 |
| sort_order | Integer | 排序順序 |
| created_at | DateTime | 建立時間 |
| updated_at | DateTime | 更新時間 |

### 4.2 菜品 (MenuItem)

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | Long | 主鍵 |
| category_id | Long | 所屬分類 |
| name | String | 菜品名稱 |
| description | String | 簡短描述 |
| price | Decimal | 價格 |
| image_url | String | 圖片路徑 |
| is_recommended | Boolean | 是否為推薦菜品 |
| status | Enum | 上架 / 下架 / 售完 |
| is_deleted | Boolean | 軟刪除標記 |
| created_at | DateTime | 建立時間 |
| updated_at | DateTime | 更新時間 |

### 4.3 桌位 (Table)

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | Long | 主鍵 |
| table_number | String | 桌號（如 A1, A2, B1） |
| status | Enum | 空桌 / 用餐中 / 待清理 |
| seated_at | DateTime | 入座時間（用於計算用餐時長） |
| qr_code_url | String | QR Code 圖片路徑 |
| created_at | DateTime | 建立時間 |
| updated_at | DateTime | 更新時間 |

### 4.4 訂單 (Order)

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | Long | 主鍵 |
| order_no | String | 訂單編號（系統生成） |
| table_id | Long | 桌位 ID |
| status | Enum | 待確認 / 準備中 / 已出餐 / 已完成 / 已取消 |
| total_amount | Decimal | 訂單總金額 |
| created_at | DateTime | 下單時間 |
| updated_at | DateTime | 更新時間 |

### 4.5 訂單品項 (OrderItem)

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | Long | 主鍵 |
| order_id | Long | 所屬訂單 |
| menu_item_id | Long | 菜品 ID |
| menu_item_name | String | 菜品名稱（快照） |
| price | Decimal | 下單時單價（快照） |
| quantity | Integer | 數量 |
| remark | String | 品項備註（如「不要香菜」） |
| subtotal | Decimal | 小計 |

### 4.6 管理員 (Admin)

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | Long | 主鍵 |
| username | String | 帳號 |
| password_hash | String | 密碼雜湊 |
| created_at | DateTime | 建立時間 |

## 5. 非功能性需求

| 編號 | 項目 | 說明 |
|------|------|------|
| PRD-NF-001 | 響應式設計 | 顧客端須適配各種手機螢幕尺寸 |
| PRD-NF-002 | 載入速度 | 顧客掃碼後頁面須在 3 秒內可操作 |
| PRD-NF-003 | 無需安裝 | 顧客端為 Mobile Web，無需下載 App |
| PRD-NF-004 | 瀏覽器相容 | 支援 iOS Safari 及 Android Chrome |
| PRD-NF-005 | 中文介面 | 系統介面預設為繁體中文 |

---

## 6. 不在範圍內 (Out of Scope)

以下功能在本階段**不納入開發**：

- 線上付款 / 結帳功能（餐廳沿用現有收銀方式）
- 會員系統 / 顧客帳號
- 多語系支援
- 廚房顯示螢幕 (KDS)
- 即時推播通知（如 WebSocket 訂單通知）
- 營收報表與統計
- 多店面管理
