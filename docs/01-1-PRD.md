# 01-1 產品需求文件 (PRD)

> **專案名稱**：Vibe Money Book — 語音記帳應用
> **版本**：v1.4
> **最後更新**：2026-03-27

---

## 1. 產品概述

Vibe Money Book 是一款「一句話記帳」應用，使用者透過語音或文字輸入自然語言描述消費行為，系統利用 LLM 自動萃取金額、類別、商家等結構化資料並記錄。與傳統記帳工具不同，本產品具備**個性化 AI 人設**（毒舌、溫柔、情勒），在記帳的同時提供具有個性的財務評論與即時預算回饋，讓記帳不再枯燥。

### 1.1 核心痛點

- 傳統記帳 App 需手動選擇分類、輸入金額，操作摩擦力高，導致使用者難以堅持
- 缺乏情緒連結，純工具式體驗容易讓人放棄

### 1.2 目標使用者

| 角色 | 說明 |
|------|------|
| 一般使用者 (User) | 希望輕鬆記帳、掌握預算的個人用戶 |

### 1.3 核心目標

- 透過自然語言（語音或文字）實現零摩擦記帳
- 利用 LLM 自動分類與結構化消費資料
- 以個性化 AI 人設提供情緒回饋，提升使用黏著度
- 即時顯示預算狀態，幫助使用者控制支出

## 2. 功能需求

### 2.1 語音與文字輸入

#### PRD-F-001：語音輸入組件

> 📐 UI 規格：[`01-4-UI_UX_Design.md` §4.1 語音錄音中](./01-4-UI_UX_Design.md#41-語音錄音中-voice-recording)

- 整合 Web Speech API，提供「按住說話」(Push-to-Talk) 的麥克風按鈕
- 按住麥克風按鈕時，顯示動態視覺回饋（聲波動畫或脈衝效果）
- 鬆開按鈕後，自動將語音轉為文字並送出
- 需支援中文語音辨識
- 語音辨識語言跟隨使用者的 UI 語言設定自動切換（詳見 PRD-F-015）
- 語音辨識失敗時，顯示友善提示並允許重試或改用文字輸入

#### PRD-F-002：文字輸入

- 提供文字輸入框，使用者可直接打字輸入消費描述
- 支援自然語言格式，如：「午餐吃拉麵 180 元」、「搭捷運 35」、「星巴克咖啡 150」
- 按 Enter 或點擊送出按鈕提交

### 2.2 儀表板式介面

#### PRD-F-003：儀表板式首頁佈局

> 📐 UI 規格：[`01-4-UI_UX_Design.md` §3.1 首頁](./01-4-UI_UX_Design.md#31-首頁-記帳主頁)

- 採用儀表板式（Dashboard）佈局設計
- 頁面由上至下依序為：預算卡片、AI 回饋卡片、最近帳目列表、底部輸入列
- 預算卡片即時顯示本月預算使用狀況（對應 PRD-F-007）
- AI 回饋卡片顯示最近一筆記帳的 AI 人設評論
- 最近帳目列表顯示近期消費記錄摘要
- 底部固定輸入列整合文字輸入框與語音輸入按鈕，實現一句話記帳

#### PRD-F-004：AI 解析結果確認

> 📐 UI 規格：[`01-4-UI_UX_Design.md` §4.2 AI 解析結果確認卡片](./01-4-UI_UX_Design.md#42-ai-解析結果確認卡片-parsed-result-card)

- LLM 解析使用者輸入後，在對話中以卡片形式顯示結構化結果：
  - 金額 (amount)
  - 類別 (category)
  - 商家/描述 (merchant)
  - 日期 (date，預設為今天)
- 使用者可點擊「確認」儲存，或「修改」進行手動調整

### 2.3 AI 人設系統

#### PRD-F-005：人設選擇與切換

> 📐 UI 規格：[`01-4-UI_UX_Design.md` §3.4 設定頁](./01-4-UI_UX_Design.md#34-設定頁-settings)（人設選擇區）、[§3.1.3 AI 回饋卡片](./01-4-UI_UX_Design.md#313-ai-回饋卡片-ai-feedback-card)（人設名稱對應）

- 提供三種預設 AI 人設：
  - **毒舌模式** 🔥：尖銳犀利的財務評論（如「650 元買壽司？你準備靠光合作用過活嗎？」）
  - **溫柔模式** 💖：溫暖鼓勵的財務陪伴（如「辛苦了～偶爾犒賞自己也很重要呢」）
  - **情勒模式** 🥺：帶有愧疚感的善意提醒（如「這個月已經超支了...我好擔心你啊...」）
- 使用者可在設定頁面選擇偏好人設
- 切換人設後，後續所有 AI 評論立即套用新風格

#### PRD-F-006：AI 情緒回饋生成

- 根據以下因素動態生成評論：
  - 當前選定的人設風格
  - 本月預算使用狀況（剩餘比例）
  - 單筆消費金額相對於預算的比例
  - 消費類別（必要支出 vs 非必要支出）
- 評論文字以對話泡泡形式呈現，附帶情緒標籤（如：鼓勵、警告、嘲諷、心疼）

### 2.4 預算管理與視覺化

#### PRD-F-007：預算血條 (Budget Progress Bar)

> 📐 UI 規格：[`01-4-UI_UX_Design.md` §3.1.2 預算卡片](./01-4-UI_UX_Design.md#312-預算卡片-budget-card)、[§4.4 預算警示](./01-4-UI_UX_Design.md#44-預算警示-budget-alert)

- 頁面頂部或固定區域顯示「本月預算血條」
- 血條顯示本月已使用預算佔總預算的百分比
- 血條顏色隨剩餘預算變化：
  - ≥ 50% 剩餘：綠色
  - 20% ~ 50% 剩餘：黃色
  - < 20% 剩餘：紅色，觸發「呼吸燈」或「閃爍警告」特效
- 顯示數字：已花費 / 總預算（如 $18,500 / $30,000）

#### PRD-F-008：消費分佈圓餅圖

> 📐 UI 規格：[`01-4-UI_UX_Design.md` §3.2 統計頁](./01-4-UI_UX_Design.md#32-統計頁-stats)

- 整合 Recharts，根據消費類別生成圓餅圖
- 顯示本月各類別消費佔比（如：飲食 40%、交通 15%、娛樂 20%...）
- 點擊圓餅圖區塊可查看該類別的消費明細
- 圓餅圖位於專屬的「統計」頁面或對話介面下方的摘要區

#### PRD-F-009：交易記錄列表

> 📐 UI 規格：[`01-4-UI_UX_Design.md` §3.3 記錄頁](./01-4-UI_UX_Design.md#33-記錄頁-history)、[§4.5 刪除操作](./01-4-UI_UX_Design.md#45-刪除操作)

- 顯示所有消費記錄，按日期倒序排列
- 每筆記錄顯示：日期、類別圖標、商家/描述、金額
- 支援按類別篩選
- 支援按日期範圍篩選（本週、本月、自訂）
- 支援語音/自然語義篩選查詢（詳見 PRD-F-014）
- 可點擊單筆記錄查看詳情（含 AI 評論、原始輸入文字）
- 支援刪除單筆記錄

#### PRD-F-014：語音/自然語義篩選查詢

- 在記錄頁面底部新增語音/文字輸入框，用以輸入自然語言的篩選條件
- 支援語音輸入（複用 PRD-F-001 的語音輸入組件）與文字輸入
- 查詢範例：
  - 「最近一個月毛小孩的開銷」
  - 「在買衣服的事上面花了多少」
  - 「阿貓阿狗這家店我花了多少錢」
  - 「潛水總共花了多少錢」
- **篩選模式互斥**：
  - 手動篩選器（日期範圍、類別下拉）與自然語言查詢互斥
  - 啟用手動篩選器時，自動清除自然語言查詢輸入
  - 輸入自然語言查詢時，自動清除手動篩選器的條件
  - 「清除篩選」按鈕一次清除所有篩選條件（含手動與自然語言）
- **AI 查詢流程**（後端處理）：
  1. 提示詞附加：當前時間與時區
  2. AI 解析使用者原始輸入中的時間範圍；若無法解析則預設為當月
  3. 依解析出的時間範圍查詢完整交易記錄（含 note 備註欄），組裝為交易記錄集
  4. 將交易記錄集附加至使用者原始查詢，再次送 AI 分析並回覆
  5. AI 回覆內容：
     - 一句總結評語（依當前人設風格，以 Chat 方式呈現，參考首頁 AI 教練區塊）
     - 匹配的交易記錄 ID 列表，用於前端篩選顯示
- **UI 呈現**：
  - 查詢結果返回後，頁面上方顯示 AI 教練回饋區塊（僅在有查詢結果時顯示）
  - 下方交易列表僅顯示 AI 標記為匹配的帳目
- **範例**：
  - 輸入：「阿貓阿狗這家店我花了多少錢」
  - 上方 AI 教練區塊：「你這個月大手大腳地在毛孩子身上花了 2800 元，是愛心氾濫了，還是你被毛孩子威脅了？」
  - 下方列出所有符合的帳目

### 2.5 多語系支援

#### PRD-F-015：i18n 多語系支援

本功能為應用程式提供完整的多語系（國際化）支援，涵蓋 UI 介面、系統訊息、預設資料與 AI 互動四大範圍。

**支援語言**：

| 語言代碼 | 語言名稱 | 說明 |
|---------|---------|------|
| `zh-TW` | 繁體中文 | 預設語言 |
| `en` | English | 第二語言 |
| `zh-CN` | 简体中文 | — |
| `vi` | Tiếng Việt | 越南文 |

**PRD-F-015-1：UI 介面多語化**

- 所有頁面的靜態文字（標題、按鈕、標籤、提示文字、佔位符）皆須支援多語切換
- 涵蓋頁面：首頁、統計頁、記錄頁、設定頁、登入頁、註冊頁
- 涵蓋元件：導航列、預算卡片、AI 回饋卡片、交易列表、圓餅圖圖例、語音輸入、解析結果卡片、新類別對話框等
- 數字格式跟隨語言設定（如英文 `1,000.00`、中文 `1,000.00`、越南文 `1.000,00`）
- 日期格式跟隨語言設定（如中文 `2026/03/22`、英文 `Mar 22, 2026`、越南文 `22/03/2026`）
- 貨幣符號跟隨語言設定顯示

**PRD-F-015-2：系統訊息多語化**

- 所有後端 API 回傳的錯誤訊息須依使用者語言回傳對應翻譯
  - 包含：參數驗證錯誤、認證錯誤、資源不存在、速率限制、伺服器錯誤等
- 前端本地的提示訊息（toast、確認對話框、表單驗證提示）須支援多語
- 後端透過 `Accept-Language` Request Header 判斷回應語言

**PRD-F-015-3：預設類別名稱多語化**

- 系統預設類別（飲食、交通、娛樂等）的顯示名稱須依語言設定顯示對應翻譯
- 類別代碼（`food`、`transport` 等）維持英文不變，僅顯示名稱多語化
- 使用者自訂類別名稱以建立時的語言儲存，不自動翻譯

**PRD-F-015-4：AI 提示詞多語化**

- **資料萃取引擎**（Data Extractor）：輸出為結構化 JSON，語言無關，但 System Prompt 須指示 LLM 能理解使用者語言的輸入
- **人設回饋引擎**（Persona Engine）：回饋文字須以使用者的語言設定生成（如英文使用者收到英文的毒舌評論）
- **聊天回覆引擎**（Chat Reply）：回覆語言跟隨使用者語言設定
- **語義查詢引擎**（Query Engine）：總結評語以使用者語言生成，交易匹配邏輯須能理解多語言查詢
- **意圖偵測引擎**（Intent Detector）：須能辨識各語言的記帳意圖與對話意圖

**語言偏好設定機制**：

1. **預設判斷**：使用者首次進入時，依瀏覽器語言設定（`navigator.language`）自動偵測並套用最接近的支援語言；若無匹配，fallback 至 `zh-TW`
2. **手動切換**：使用者可在設定頁手動選擇語言，選擇後立即生效，無需重新整理
3. **持久化**：
   - 已登入使用者：語言偏好儲存至 User 資料（DB），跨裝置同步
   - 未登入使用者：語言偏好儲存於 `localStorage`，登入後自動同步至 DB
4. **語音辨識連動**：UI 語言切換時，Web Speech API 的 `recognition.lang` 同步切換至對應語言代碼

**語言代碼與語音辨識對應**：

| UI 語言 | 語音辨識語言代碼 |
|---------|----------------|
| `zh-TW` | `zh-TW` |
| `en` | `en-US` |
| `zh-CN` | `zh-CN` |
| `vi` | `vi-VN` |

### 2.6 統計頁面交互

#### PRD-F-016：統計頁面條狀圖交互展開

- 統計頁面的類別消費條狀圖支援點擊展開/收合交互：
  - **展開**：點擊某類別的條狀圖，在該條下方展開該類別的交易記錄列表
  - **記錄明細**：點擊列表中的某筆記錄，展開該筆記錄的詳細資訊（金額、商家、日期、備註）
  - **收合**：再次點擊同一條狀圖，收合已展開的交易記錄列表
  - 同一時間僅展開一個類別（點擊另一類別時自動收合前一個）
- 展開的交易記錄列表按日期倒序排列
- 列表中每筆記錄顯示：日期、商家/描述、金額
- 展開/收合動畫應流暢（建議使用 CSS transition 或 framer-motion）

### 2.7 設定功能

#### PRD-F-010：使用者設定

> 📐 UI 規格：[`01-4-UI_UX_Design.md` §3.4 設定頁](./01-4-UI_UX_Design.md#34-設定頁-settings)

- 設定每月總預算金額
- 選擇 AI 人設偏好（毒舌/溫柔/情勒）
- 設定預設消費幣別（初期僅支援 TWD）
- 選擇介面語言（詳見 PRD-F-015）

#### PRD-F-011：類別預算設定

- 預設消費類別：飲食、交通、娛樂、購物、日用品、醫療、教育、其他
- 可為每個類別設定個別預算限額
- 類別預算總和不需等於月總預算（個別類別限額為獨立警示用途）
- 超過類別限額時，AI 評論會特別提醒

#### PRD-F-012：AI 自動偵測並新增消費類別

> 📐 UI 規格：[`01-4-UI_UX_Design.md` §5 PRD-F-012 新類別對話流 UI](./01-4-UI_UX_Design.md#5-prd-f-012-新類別對話流-ui)

- 使用者透過文字或語音輸入消費時，LLM 萃取類別後與現有類別比對
- 若 AI 判斷消費內容無法適當歸入任何現有類別，應以對話方式詢問使用者：
  - 提示訊息範例：「我覺得這筆消費屬於「寵物」，但你的類別中沒有這項，要新增嗎？」
  - 使用者可確認（新增類別並完成記帳）、修改類別名稱後確認，或手動選擇現有類別
- 使用者確認後，系統新增該類別並自動套用至本筆消費記錄
- 新增的類別可在設定頁設定個別預算限額（同 PRD-F-011）
- 避免重複詢問：相似類別名稱（如「咖啡」與「飲料」）應由 AI 提示合併，而非重複建立

#### PRD-F-013：AI 引擎選擇（基礎版，已由 PRD-F-017 擴展）

> 📐 UI 規格：[`01-4-UI_UX_Design.md` §6 PRD-F-013 AI 引擎選擇 UI](./01-4-UI_UX_Design.md#6-prd-f-013-ai-引擎選擇-ui)

- 使用者可在設定頁選擇偏好的 AI 引擎：
  - **Gemini**（Google，預設）
  - **OpenAI**（GPT-4o-mini）
- 切換引擎後，資料萃取引擎與情緒生成引擎皆使用所選引擎
- 需提供 API Key 輸入欄位（金鑰儲存於本地，不上傳伺服器）
- 切換引擎即時生效，無需重新整理頁面
- 若 API Key 無效或超過 quota，以友善錯誤訊息提示使用者，引導檢查 Key 或切換引擎

> ⚠️ **M7 擴展**：PRD-F-017 將此功能擴展為多供應商 + 模型選擇架構，詳見下方 PRD-F-017。

#### PRD-F-017：AI 引擎進階設定（擴展 PRD-F-013）

本功能將 PRD-F-013 的雙引擎架構擴展為多供應商 + 模型選擇架構，並加入 API Key 測試驗證與預設引擎後備機制。

**支援的 AI 模型供應商**：

| 供應商 | 代碼 | 說明 |
|--------|------|------|
| Google Gemini | `gemini` | 預設供應商 |
| OpenAI | `openai` | GPT 系列 |
| Anthropic | `anthropic` | Claude 系列 |
| xAI | `xai` | Grok 系列 |

**功能需求**：

1. **供應商選擇**：
   - 設定頁面顯示所有支援的 AI 模型供應商列表（卡片式或列表式 UI）
   - 使用者點擊選擇偏好的供應商
   - 選擇後即時生效，無需重新整理頁面

2. **API Key 設定與測試**：
   - 每個供應商提供獨立的 API Key 輸入欄位（選填）
   - API Key 儲存於前端 `localStorage`，**不上傳至伺服器持久化**
   - 提供「測試連線」按鈕：呼叫後端 API 驗證 Key 有效性
   - 測試成功：顯示綠色成功提示（含模型回應驗證）
   - 測試失敗：顯示紅色錯誤提示（含具體失敗原因：格式錯誤、無效、配額耗盡等）
   - 儲存時自動觸發測試，測試通過才允許儲存

3. **模型選擇**：
   - API Key 設定完成（或伺服器有預設 Key 可用）後，顯示該供應商的可用模型列表
   - 模型列表由後端 API 動態取得：當使用者提供有效 API Key 時，透過供應商 API 即時查詢可用模型清單
   - 若無 API Key 或 Key 無效，則退回顯示系統維護的預設模型清單（curated defaults）
   - 使用者可從列表中選擇具體模型
   - 預設模型由系統指定（每個供應商有一個預設推薦模型）

4. **預設引擎後備機制**（`.env` 配置）：
   - 伺服器端可透過 `.env` 設定各供應商的預設 API Key
   - 當使用者未設定 API Key 時，自動使用伺服器預設的 API Key（若有配置）
   - 前端透過 `GET /ai/config` 查詢伺服器是否有預設 Key 可用
   - 若伺服器有預設 Key，設定頁面顯示「使用預設引擎」選項，無需使用者提供 Key
   - 若伺服器無預設 Key 且使用者未設定 Key，提示使用者需先設定 API Key 才能使用 AI 功能

**UI 互動流程**：

```
設定頁 → AI 引擎設定區塊
  ├── 1. 選擇供應商（Gemini / OpenAI / Anthropic / xAI）
  ├── 2. 輸入 API Key（選填，若伺服器無預設 Key 則必填）
  ├── 3. 點擊「測試連線」驗證 Key
  ├── 4. 選擇模型（模型列表根據 API Key 動態取得；無 Key 時顯示預設清單）
  └── 5. 測試通過 → 儲存設定
```

**資料模型變更**：

| 欄位 | 變更 | 說明 |
|------|------|------|
| `ai_engine` | 擴展 Enum | 新增 `anthropic`、`xai` 選項 |
| `ai_model` | **新增** String | 使用者選擇的具體模型名稱（如 `gemini-2.5-flash`、`claude-sonnet-4-5-20250514`） |

## 7. 不在範圍內 (Out of Scope)

以下功能在本階段**不納入開發**：

- 多人共享帳本 / 分帳功能
- 銀行帳戶串接 / 自動匯入交易
- 收據 OCR 掃描
- 多幣別即時匯率轉換
- 原生 App（iOS/Android）— 僅做 PWA
- 自訂 AI 人設（僅提供三種預設）
- 匯出報表（CSV/PDF）
- 新手引導 / Onboarding 流程
